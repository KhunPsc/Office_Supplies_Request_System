# ข้อเสนอแนะในการปรับปรุง Frontend - ระบบแจ้งคำขอซื้อวัสดุสำนักงาน

เอกสารฉบับนี้สรุปแนวทางการปรับปรุงระบบตามมาตรฐานความปลอดภัย (Security), โครงสร้างโค้ด (Maintainability), ประสิทธิภาพ (Performance) และประสบการณ์ผู้ใช้งาน (UX/UI)

---

## 1. ความปลอดภัย (Security) 🛡️
*   **[Critical] ย้ายระบบ Authentication:** ปัจจุบัน `userMapping` อยู่ใน `script.js` ซึ่งไม่ปลอดภัย (View Source เห็นได้หมด)
    *   **แนวทาง:** ให้ส่ง Username ไปตรวจสอบที่ Google Apps Script (Backend) และให้ Backend เป็นผู้ยืนยันสิทธิ์ (Role) กลับมา
*   **ป้องกัน XSS:** ตรวจสอบการใช้ `.innerHTML` ในส่วนที่แสดงข้อมูลจากผู้ใช้งาน
    *   **แนวทาง:** ใช้ `.textContent` หรือทำ Sanitize ข้อมูลก่อนแสดงผลเสมอ

## 2. โครงสร้างโค้ด (Maintainability) 🏗️
*   **Modularization:** แยกไฟล์ `script.js` ที่มีขนาดใหญ่เกินไปออกเป็นส่วนๆ
    *   `api.js`: สำหรับจัดการการส่งข้อมูล (Fetch)
    *   `ui-render.js`: สำหรับจัดการการแสดงผล UI/DOM
    *   `app.js`: สำหรับควบคุม Logic หลักของแอป
*   **Template System:** ลดการเขียน HTML String ใน JavaScript
    *   **แนวทาง:** ใช้ HTML `<template>` tags ใน `index.html` แล้วใช้ JS Clone แทน

## 3. ประสิทธิภาพ (Performance) ⚡
*   **Caching Data:** ปรับปรุงระบบการดึงข้อมูล `fetchTrackingData()`
    *   **แนวทาง:** ไม่ควรดึงข้อมูลใหม่ทุกครั้งที่สลับ Tab หากข้อมูลไม่มีการเปลี่ยนแปลง
*   **File Upload:** ตรวจสอบขนาดไฟล์ก่อนอัปโหลดฝั่ง Client
    *   **แนวทาง:** เพิ่มระบบ Resize รูปภาพก่อนส่ง (ถ้าเป็นรูปภาพ) เพื่อลดภาระการส่งข้อมูล Base64

## 4. ประสบการณ์ผู้ใช้งาน (UX/UI) 📱
*   **Toast Notifications:** เพิ่มระบบแจ้งเตือนที่ชัดเจนเมื่อบันทึกข้อมูลสำเร็จ (แทนการเปลี่ยนข้อความในปุ่ม)
*   **Mobile-Friendly Tables:** ปรับแต่ง CSS สำหรับตารางในหน้าจอมือถือ
    *   **แนวทาง:** ใช้ Media Queries เปลี่ยนรูปแบบจากตาราง (Table) เป็นการ์ด (Cards) เมื่อหน้าจอเล็กกว่า 768px
*   **Loading States:** เพิ่ม Skeleton Screen หรือ Spinner ที่สวยงามในทุกจุดที่มีการรอข้อมูล

## 5. การปรับปรุง .gitignore ⚙️
*   ควรยกเว้นไฟล์ที่มีข้อมูลการตั้งค่าเฉพาะตัว หรือไฟล์บันทึกข้อเสนอแนะที่ไม่ได้เป็นส่วนหนึ่งของระบบจริงออกจาก Repository
