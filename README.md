# 📦 ระบบแจ้งคำขอซื้อวัสดุสำนักงาน PEA จอมทอง

Web Application สำหรับอำนวยความสะดวกในการแจ้งคำขอซื้อวัสดุสำนักงานและติดตามสถานะงาน สำหรับพนักงานและเจ้าหน้าที่ที่เกี่ยวข้อง

## ✨ ฟีเจอร์หลัก
- 📝 **User Request Form:** ฟอร์มขอซื้อที่กรอกง่าย รองรับการเพิ่ม/ลบรายการวัสดุและแนบไฟล์
- ⚡ **Quick Select:** วัสดุยอดนิยมแบบคลิกเพิ่มทันที, รูปสินค้าแสดงแบบสี่เหลี่ยม, มีระบบป้องกันเพิ่มซ้ำ
- 🔍 **Tracking & History:** ติดตามสถานะคำขอแบบเรียลไทม์ และดูประวัติคำขอที่เสร็จสิ้น/ยกเลิก
- 📊 **Admin Workboard:** มุมมองผู้ดูแลสำหรับคัดแยก, เปลี่ยนสถานะ, แก้ไขจำนวน/หน่วย, และจัดการแบบกลุ่ม
- 🖨️ **Procurement Worksheet:** สร้างใบรายการจัดซื้อสำหรับพิมพ์/PDF พร้อมช่องลายเซ็น และการจัดข้อความในเซลล์แบบ auto-fit
- 🔐 **Role Guard:** ป้องกันผู้ใช้ที่ไม่มีสิทธิ์เข้ามุมมอง Admin แม้มี session ก่อนหน้า

## 🆕 อัปเดตล่าสุด (UI/UX Baseline V1)
- ปรับโทนหน้าจอ **Admin** ให้แยกจาก **User** ชัดเจน พร้อม toast แจ้งตอนสลับโหมด
- ปรับฟอร์มผู้ใช้ให้ compact มากขึ้น (ขนาดรายการ, ปุ่มเพิ่มรายการ, ปุ่มลบแบบ `x`)
- ปรับ Quick Select ให้ใช้งานต่อเนื่องขึ้น (scroll ภายใน, รูปอ่านง่าย, แสดง/ซ่อนรายการตามที่เลือกไปแล้ว)
- ปรับตารางติดตามสถานะให้อ่านง่ายขึ้น (สีหัวตาราง/เส้น grid/การจัดวางใน admin)
- ปรับ modal แสดงไฟล์ให้ปิดได้ด้วยการคลิกนอก popup
- แก้ปัญหาหน้าประวัติค้างที่สถานะ "กำลังโหลดข้อมูลประวัติ..."

## 🛠️ เทคโนโลยีที่ใช้
- Frontend: Vanilla HTML5, CSS3, JavaScript (ES6)
- Backend: Google Apps Script (GAS)
- Database: Google Sheets
- Storage: Google Drive API (สำหรับไฟล์แนบ/รูปภาพ)

## 🏗️ โครงสร้างโปรเจกต์
- `docs/` frontend หลัก (`index.html`, `style.css`, `script.js`, `config.js`)
- `backend/` Google Apps Script (`Code.gs`)
- `UI_PERFECT_V1.md` baseline อ้างอิง UI เวอร์ชันที่เสถียร
- `DESIGN.md` และ `FRONTEND_IMPROVEMENTS.md` เอกสารแนวทางดีไซน์/การพัฒนา

## 🚀 เริ่มต้นใช้งาน
1. ตั้งค่า URL ของ Apps Script ใน `docs/config.js`
2. ตรวจสอบคอลัมน์ใน Google Sheets ให้ตรงกับที่ `backend/Code.gs` ใช้งาน
3. เปิด `docs/index.html` ในเบราว์เซอร์เพื่อใช้งานระบบ
